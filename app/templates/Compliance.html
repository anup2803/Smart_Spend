Compliance

Data Security

  • Password encryption (e.g., bcrypt/sha256).
  • Session handling or JWT tokens for authentication.
  • SQL injection and CSRF protection in backend code.

Privacy Principles

  • Follows data minimization (only required user data is collected).
  • Ensures user consent before account creation.

Best Practices Alignment

While not yet a commercial app, Smart Spend aligns with common international standards such as:

  • GDPR principles (user data rights, consent, deletion).
  • OWASP security guidelines (to reduce vulnerabilities).
